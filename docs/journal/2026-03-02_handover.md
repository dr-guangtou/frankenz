## Problems / Blockers
- **NumPy 2.0 incompatibility**: Library code uses `np.trapz` (removed in NumPy 2.0). Causes test failures in `simulate.py` and `plotting.py`. The example script uses `np.trapezoid` correctly.
- **PDF under-dispersion**: zerr clamping to 0.01 floor produces over-confident PDFs (36% coverage at nominal 68%). Proper fix would require adaptive bandwidth or better sentinel handling.
- **40% outlier rate (best estimator)**: Physically reasonable for 5-band HSC data, but improvable with data-driven KNN prior (future phase).

## Key Decisions
- **Single file, not library module**: The script is an example, not reusable infra. Well-sectioned single file is easier to share.
- **zerr clamping strategy**: Floor=0.01, Ceil=1.0. Floor matches old `ZSMOOTH=0.01` config. Ceil=1.0 catches 99.0 sentinel values. 3,128 values clamped per file.
- **`io.BytesIO` for HDF5 header**: Both HDF5 files have a 958-byte YAML config prepended. Auto-detect via scanning for `\x89HDF\r\n\x1a\n` magic bytes.
- **Manual PhotoData construction**: Can't use `frankenz.io.load_data()` because HDF5 schema is frankenz4DESI-specific (cmodel/ groups, aperture corrections).
- **Errors NOT aperture-corrected**: Following frankenz4DESI convention exactly — only flux is corrected, errors remain in raw units.
