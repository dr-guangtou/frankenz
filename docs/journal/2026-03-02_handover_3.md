## Completed This Session
- Committed example script + config to feature branch, merged to `main` (4512217)
- Fixed NumPy 2.0 `np.trapz` compat with try/except shim (getattr approach fails on NumPy 2.x because Python eagerly evaluates the fallback)
- **Diagnosed and fixed the KDE bandwidth bug** (59d947d):
  - Root cause: Our script passed raw `zerr` values as KDE bandwidth. 31% of HSC zerr values are sentinels (-9, 0, 99), clamped to [0.01, 1.0]. Bandwidth of 1.0 produces massively over-smoothed PDFs.
  - The old frankenz4DESI uses `ENABLE_ZERR=false` + `ZSMOOTH=0.01` → fixed bandwidth of 0.01 for ALL objects
  - Fix: replaced zerr-based bandwidth with constant `ZSMOOTH=0.01` in the script
  - Results improved: best estimator sigma_NMAD 0.134→0.067, f_out 0.396→0.204
